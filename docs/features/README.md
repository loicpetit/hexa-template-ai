# Feature Documentation

This folder contains user guide documentation for each product functionality, maintained by the Product Owner agent.

## Structure
Each functionality has its own subdirectory with documentation files:
- `<functionality>/FEATURE-GUIDE.md` — User guide with examples and workflows
- `<functionality>/QUICK-START.md` — Quick start guide for new users
- `<functionality>/FAQ.md` — Frequently asked questions (optional)

## Required baseline files
- FEATURE-GUIDE.md for each approved functionality

## Rules
- Keep this documentation aligned with approved user stories in user-stories/.
- Documentation must be written in **plain business language**, free of technical jargon.
- Include **real-world examples** showing how users interact with the feature.
- Include **screenshots or diagrams** (Mermaid where appropriate) to illustrate workflows.
- Link to related user stories and requirements in metadata.
- Update feature docs whenever related user stories are created, replaced, or approved.
- Use a consistent format: Overview → Key Concepts → How-To Guides → Examples → Troubleshooting/FAQ.
